# React Beginner

> 현재 학습 단계: 초급(Beginner)

퍼블리싱 경험을 기반으로 JavaScript와 React를 학습하고, 직접 동작하는 UI를 만들 수 있는 프론트엔드 개발자로 성장하기 위한 학습 저장소입니다.

HTML과 CSS로 화면을 구현하는 능력을 React의 컴포넌트, Props, State 개념과 연결하는 것을 목표로 합니다.

강의나 예제 코드를 그대로 따라 작성하는 데서 끝내지 않고, 각 단계에서 다음 과정을 반복합니다.

1. 동작을 예상한다.
2. 작은 기능을 직접 구현한다.
3. 오류와 예외 상황을 확인한다.
4. 코드를 다시 정리한다.
5. 배운 점과 어려웠던 점을 기록한다.

---

## 누구를 위한 과정인가요?

다음과 같은 경험이 있는 퍼블리셔를 대상으로 합니다.

- HTML과 CSS로 웹페이지를 구현할 수 있다.
- 반응형 레이아웃을 작성할 수 있다.
- 탭, 모달, 아코디언 등의 UI를 구현해본 경험이 있다.
- JavaScript 또는 jQuery 코드를 일부 읽어본 경험이 있다.
- 정적인 화면뿐 아니라 데이터에 따라 변경되는 화면을 만들고 싶다.

JavaScript를 완벽하게 알고 있을 필요는 없습니다. 필요한 문법을 먼저 익힌 후 React 실습을 진행하고, 부족한 내용은 실습 과정에서 다시 학습합니다.

## 퍼블리싱 경험과 React 연결하기

| 퍼블리싱에서 사용한 개념 | React에서 배우는 개념 |
|---|---|
| 반복되는 HTML 구조 | 컴포넌트 |
| 클래스에 따른 화면 변화 | Props와 State |
| 목록 마크업 | 배열과 `map` |
| 버튼 클릭 이벤트 | 이벤트 핸들러 |
| 탭과 아코디언 | 조건부 렌더링 |
| 입력 폼 | Controlled Component |
| 공통 마크업 | 컴포넌트 합성 |
| API 데이터 출력 | Effect와 서버 상태 |

## 초급 과정의 범위

초급 과정에서는 다음 능력을 갖추는 데 집중합니다.

- JavaScript로 배열과 객체 데이터를 다룬다.
- UI를 React 컴포넌트로 분리한다.
- Props와 State를 사용해 화면을 변경한다.
- 사용자 입력과 이벤트를 처리한다.
- API 데이터를 화면에 출력한다.
- TypeScript로 기본적인 데이터와 Props 타입을 작성한다.
- 로딩, 빈 상태, 오류 상태를 구분한다.
- 키보드와 스크린 리더를 고려한 UI를 작성한다.
- 구현 결과와 검증 방법을 설명한다.

복잡한 상태관리 라이브러리나 프레임워크를 먼저 배우기보다 JavaScript와 React의 기본 동작을 이해하는 데 집중합니다.

## React를 시작하기 전에 알아야 할 것

React는 JavaScript를 사용해 UI를 만드는 라이브러리입니다.

React 코드를 이해하려면 JavaScript의 함수, 배열, 객체와 이벤트 처리 방법을 알아야 합니다. TypeScript를 사용하는 프로젝트에서는 기본적인 타입 표현도 읽을 수 있어야 합니다.

JavaScript와 TypeScript를 모두 완벽하게 학습한 후 React를 시작할 필요는 없습니다. 아래 기능을 문서를 참고하면서 구현할 수 있다면 React 학습을 시작하기에 충분합니다.

아래 기능은 HTML과 JavaScript로 구현하거나, 화면 구현이 어렵다면 배열을 조작한 결과를 콘솔에 출력해도 됩니다.

1. 입력창에서 문자열을 입력받는다.
2. 버튼을 누르면 입력값을 배열에 추가한다.
3. 배열을 순회해 목록으로 출력한다.
4. 특정 항목을 삭제한다.
5. 데이터에 간단한 TypeScript 타입을 지정한다.

구현이 어렵다면 React Hook을 먼저 외우기보다 함수, 배열, 객체와 이벤트 처리부터 복습합니다.

## 학습 방법

각 주제는 다음 순서로 학습합니다.

### 1. 예상하기

코드를 작성하기 전에 어떤 입력을 받으며 화면이 어떻게 변경될지 적습니다.

### 2. 최소 기능 구현하기

한 번에 많은 기능을 만들지 않고, 한 가지 동작만 먼저 구현합니다.

### 3. 직접 다시 작성하기

완성된 예제를 본 후 예제를 닫고 같은 기능을 다시 작성합니다.

### 4. 요구사항 추가하기

기본 기능이 동작하면 새로운 조건을 하나 추가합니다.

예시:

- 빈 값은 등록할 수 없다.
- 같은 이름은 중복 등록할 수 없다.
- 삭제 전에 확인 메시지를 표시한다.
- 결과가 없으면 빈 상태 화면을 표시한다.

### 5. 검증하기

정상 동작뿐 아니라 다음 상황도 확인합니다.

- 입력값이 없는 경우
- 데이터가 없는 경우
- 요청이 실패한 경우
- 사용자가 빠르게 여러 번 클릭한 경우
- 키보드만 사용하는 경우

### 6. 기록하기

실습을 마치면 다음 내용을 기록합니다.

- 새로 알게 된 점
- 이해하기 어려웠던 점
- 발생한 오류와 해결 방법
- 다시 작성한다면 개선할 부분

## 실습 환경 (react-app)

React 실습은 저장소에 포함된 `react-app/` 폴더에서 진행합니다. 최신 스택으로 미리 세팅되어 있어 클론 후 바로 시작할 수 있습니다.

- **React 19 + TypeScript** — 컴포넌트와 타입
- **Vite** — 개발 서버와 빌드
- **Tailwind CSS v4** — 유틸리티 클래스 기반 스타일

빠른 시작:

```bash
cd react-app
pnpm install   # 최초 1회 (pnpm이 없으면 npm install -g pnpm)
pnpm dev       # http://localhost:3000
```

실행 방법, 스크립트, 폴더 구조와 실행 흐름은 [react-app/README.md](react-app/README.md)에 자세히 정리되어 있습니다.

## 실습 완료 체크리스트

각 실습을 완료할 때 다음 내용을 확인합니다.

- [ ] 요구사항을 먼저 작성했다.
- [ ] 정상 동작을 확인했다.
- [ ] 빈 상태를 확인했다.
- [ ] 오류 또는 잘못된 입력을 확인했다.
- [ ] 컴포넌트의 역할을 설명할 수 있다.
- [ ] 불필요하게 중복된 State가 없는지 확인했다.
- [ ] 키보드만으로 주요 기능을 사용할 수 있다.
- [ ] 배운 점과 어려웠던 점을 기록했다.
- [ ] 의미 있는 단위로 커밋했다.

## Commit Convention

```text
docs: 학습 계획과 회고 작성
feat: 새로운 실습 기능 추가
test: 테스트 추가 또는 수정
refactor: 동작 변경 없는 구조 개선
fix: 잘못된 동작 수정
style: 마크업 또는 스타일 수정
```

### 커밋 예시

```text
feat: 할 일 등록 기능 구현
fix: 빈 값이 등록되는 문제 수정
test: 할 일 삭제 사용자 흐름 추가
docs: State 학습 회고 작성
```
